# PRD：智能优化 Prompt

> Prompt Loop fork 产品需求。对齐扣子罗盘「[智能优化提示词](https://loop.coze.cn/open/docs/cozeloop/optimize-prompts-with-ai)」的产品形态；算法细节见 [TRD](./trd-smart-prompt-optimize.md)。

## 1. 背景与目标

### 1.1 目标

在 Prompt 开发页发起「智能优化」任务：基于评测证据（Goodcase / Badcase）自动迭代 Prompt，产出前后对比报告，并支持采纳为新版本。

### 1.2 非目标

- 不替代人工最终决策（采纳需用户确认并提交版本）
- MVP 不支持 Jinja2、多模态变量、无 String 变量的 Prompt
- 不做编辑器内「一键润色系统词」（那是另一条能力，无评测闭环）

## 2. 用户故事

1. 有成功实验 → 选低分 Badcase → 优化 Prompt → 查看得分提升 → 提交新版本
2. 有优质评测集 → 选 Goodcase → 优化 Prompt → 查看与参考答案贴近程度 → 提交新版本

## 3. UI 入口（对齐商业版）

Prompt 开发页 Header 操作区从左到右：

1. 对比调试
2. 版本记录
3. **智能优化**（主色实心按钮 + 下拉箭头）
4. 提交新版

「智能优化」下拉两项：

| 选项 | 说明 |
|------|------|
| 基于评测实验优化 Prompt | 根据批量评测的问题、模型回答、参考回答（如有）以及对应评估器得分来优化 Prompt |
| 基于优质的评测集优化 Prompt | 适合仅有批量问题、模型回答与参考回答的场景；据此优化 Prompt |

二级入口：详情页 Tab 行增加 **智能优化**（与「编排」并列），用于任务列表与报告。

权限：`pe.prompt.smart_optimize`（开源默认放行）。

## 4. 核心流程

```text
Header 下拉选路径
  → 选择实验 / 评测集版本
  → 选择 10–500 条样本
  → 配置字段映射 + 优化模式（效果优先 ↔ 性价比优先）
  → 提交优化任务
  → 查看报告（Prompt Diff / 分数对比 / 明细）
  → 采纳 → 写入草稿 → 用户点击「提交新版」
```

## 5. 功能需求

| ID | 需求 | 验收 |
|----|------|------|
| P1 | Header「智能优化」位于版本记录与提交新版之间，带下拉两路径 | 位置与商业版一致；无权限隐藏 |
| P2 | 下拉两项：基于评测实验 / 基于优质评测集 | 文案对齐商业版 |
| P3 | 样本选择：10–500 条；实验路径须存在非满分；须能映射参考输出 | 不满足不可进入下一步 |
| P4 | 映射：Prompt 变量 / 模型回答 / 参考回答 ↔ 评测字段 | 与实验创建映射语义一致 |
| P5 | 优化模式滑块：效果优先 ↔ 性价比优先 + 预估消耗展示 | 提交前可见 |
| P6 | Tab「智能优化」：任务列表、详情、报告 | 状态：排队/运行/成功/失败/已终止 |
| P7 | 报告：Prompt 前后对比、综合得分分布、（实验路径）评估器得分明细 | 可查看样本行 |
| P8 | 采纳成功结果 → 写入编辑草稿并引导提交新版本 | 新版本内容 = 优化后 Prompt |

## 6. 字段语义

| 产品语义 | 字段名 |
|----------|--------|
| 预期/参考输出 | `reference_output` |
| 模型真实回复 | `actual_output` |

商业文档中若将参考输出写成 `actual_output`，以本表为准。

## 7. 限制

- 不支持 Jinja2 模板引擎的 Prompt
- 不支持多模态变量的 Prompt（MVP）
- Prompt 须至少包含一个 String 类型变量
- 样本量 10–500
- 实验路径：实验须成功完成，且所选样本中存在非满分项

## 8. 与「一键优化」的区分

| 能力 | 输入 | 闭环 |
|------|------|------|
| 一键优化 | 当前系统词文本 | 无评测再评估 |
| 智能优化（本能力） | 评测集 / 实验证据 | 改写 → 再评估 → 选优 |

## 9. 成功指标

- 任务完成率
- 采纳率（采纳 / 成功任务）
- 优化前后平均分提升
- 单次任务耗时与 Token 成本

## 10. 里程碑

| 阶段 | 交付 |

| M1 | PRD + TRD |
| M2 | 前端入口 + 向导 + Tab（Mock API） |
