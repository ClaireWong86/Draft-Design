# PRD：智能优化 Prompt

> Prompt Loop fork 产品需求。对齐扣子罗盘「[智能优化提示词](https://loop.coze.cn/open/docs/cozeloop/optimize-prompts-with-ai)」的产品形态；算法细节见 [TRD](./trd-smart-prompt-optimize.md)。

## 1. 背景与目标

### 1.1 目标

在 Prompt 开发页发起「智能优化」任务：基于评测证据（Goodcase / Badcase）自动迭代 Prompt，产出前后对比报告，并支持采纳为新版本。

### 1.2 非目标

- 不替代人工最终决策（采纳需用户确认并提交版本）
- MVP 不支持 Jinja2、图像侧自动变换、无可映射变量的 Prompt；支持多模态样本与媒体内容透传
- 不做编辑器内「一键润色系统词」（那是另一条能力，无评测闭环）

## 2. 用户故事

1. 有成功实验 → 选低分 Badcase → 优化 Prompt → 查看得分提升 → 提交新版本
2. 有优质评测集 → 选 Goodcase → 优化 Prompt → 查看与参考答案贴近程度 → 提交新版本

## 3. UI 入口（对齐商业版）

Prompt 开发页 Header 操作区从左到右：

1. 对比调试
2. 版本记录
3. **智能优化**（主色实心按钮 + 下拉箭头）
4. 提交新版

「智能优化」下拉两项：

| 选项 | 说明 |
|------|------|
| 基于评测实验优化 Prompt | 根据批量评测的问题、模型回答、参考回答（如有）以及对应评估器得分来优化 Prompt |
| 基于优质的评测集优化 Prompt | 适合仅有批量问题、模型回答与参考回答的场景；据此优化 Prompt |

Prompt 详情页工作台固定为四类能力：**编排、观测、评测、智能优化**。智能优化用于任务列表与报告；编排仍负责 Prompt 内容、变量和模型配置，观测与评测继续提供生产证据和离线实验数据。

权限：`pe.prompt.smart_optimize`（开源默认放行）。

## 4. 核心流程

```text
Prompt 开发页 Header 或实验详情页「智能优化」入口
  → 选择数据来源（评测实验 / Goodcase 评测集）并确认 Prompt 版本
  → 进入独立创建页
  → 选择可用数据明细
  → 配置字段映射 + 优化模式（效果优先 ↔ 性价比优先）
  → 提交优化任务
  → Goodcase：冻结原 Prompt 基线，同裁判评估 baseline 与候选
  → 查看报告（Prompt Diff / 分数对比 / 明细）
  → 仅在验证集取得正向增益时推荐 → 采纳 → 写入草稿 → 用户点击「提交新版」
```

## 5. 功能需求

| ID | 需求 | 验收 |
|----|------|------|
| P1 | Header「智能优化」位于版本记录与提交新版之间，带下拉两路径 | 位置与商业版一致；无权限隐藏 |
| P2 | 下拉两项：基于评测实验 / 基于优质评测集 | 文案对齐商业版 |
| P3 | 实验路径选择 10–500 条；Goodcase 路径允许从 1 条开始验证；须能映射参考输出 | 实验路径还须存在非满分项，不满足不可进入下一步 |
| P4 | 映射：Prompt 变量 / 模型回答 / 参考回答 ↔ 评测字段 | 与实验创建映射语义一致 |
| P5 | 优化模式滑块：效果优先 ↔ 性价比优先 + 预估消耗展示 | 提交前可见 |
| P6 | Tab「智能优化」：任务列表、详情、报告 | 状态：排队/运行/成功/失败/已终止 |
| P7 | 报告：Prompt 前后对比、综合得分分布、（实验路径）评估器得分明细 | 可查看样本行 |
| P8 | 采纳成功结果 → 写入编辑草稿并引导提交新版本 | 新版本内容 = 优化后 Prompt |
| P9 | 实验详情页提供「智能优化」入口并回填当前实验、Prompt 与版本 | 仅成功的 Prompt 实验可进入样本选择 |
| P10 | 多模态样本保留图片/视频输入，优化前后保留完整消息与 ContentPart | 不因优化丢失媒体、角色或 Few-shot |
| P11 | 新建流程采用确认弹窗 + 独立两步创建页 | 第一步选择数据明细；第二步配置映射与优化模式；取消或完成后回任务列表 |
| P12 | 智能优化任务台支持按名称搜索、状态筛选 | 展示任务名、状态、数据条数、优化模式、结果、关联评测集/实验和操作 |
| P13 | Goodcase 计算原 Prompt 基线，并展示候选绝对分、基线分和验证集增益 | 无正向增益时任务明确拒绝推荐 |
| P14 | 原 Prompt 与候选复用同一裁判模型、评判规则和样本快照 | 结构化硬约束不得完全依赖 LLM 裁判 |

## 6. 字段语义

| 产品语义 | 字段名 |
|----------|--------|
| 预期/参考输出 | `reference_output` |
| 模型真实回复 | `actual_output` |

商业文档中若将参考输出写成 `actual_output`，以本表为准。

## 7. 限制

- 不支持 Jinja2 模板引擎的 Prompt
- 支持图片/视频/多段内容透传；MVP 不自动裁剪、画框、生成 SoM 或改写媒体
- Prompt 须至少包含一个可与评测集字段映射的变量
- 实验路径样本量 10–500；Goodcase 支持小样本验证，生产优化仍建议使用代表性验证集
- 实验路径：实验须成功完成，且所选样本中存在非满分项

## 8. 与「一键优化」的区分

| 能力 | 输入 | 闭环 |
|------|------|------|
| 一键优化 | 当前系统词文本 | 无评测再评估 |
| 智能优化（本能力） | 评测集 / 实验证据 | 改写 → 再评估 → 选优 |

## 9. 成功指标

- 任务完成率
- 采纳率（采纳 / 成功任务）
- 优化前后平均分提升
- 无提升候选拒绝率与错误推荐率
- 单次任务耗时与 Token 成本
